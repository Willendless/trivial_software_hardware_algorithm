# 一些软硬件算法的模拟与实现

## TAPSA

单个处理器基于表的优先级计算与排序。

输入文件格式

```c
5 // 5个任务
1 0 // 任务1执行时间为1，释放时间为0
1 3 // 任务2
2 0 // 任务3
1 0 // 任务4
1 0 // 任务5
1 4 // 依赖关系：1->4
1 5 // 依赖关系：1->5
2 4 // 依赖关系: 2->4
2 5 // 依赖关系: 2->5
3 5 // 依赖关系: 3->5
```

## MUPSA

在TAPSA计算得到的优先级基础上，对多个处理器进行优先级调度的模拟。本实现支持非抢占式和抢占式两种配置。

输入文件格式

```c
0   // 0表示非抢占式，1表示抢占式
5 2 // 5个任务，2个cpu
1 0 // 任务1执行时间为1，释放时间为0
1 3 // 任务2
2 0 // 任务3
1 0 // 任务4
1 0 // 任务5
1 4 // 依赖关系：1->4
1 5 // 依赖关系：1->5
2 4 // 依赖关系: 2->4
2 5 // 依赖关系: 2->5
3 5 // 依赖关系: 3->5
```
